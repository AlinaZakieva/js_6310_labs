# ShoeStore Bot

Телеграм-бот-консультант для интернет-магазина обуви ShoeStore.

---

## Команды бота

### `/start`

Краткая справка по возможностям бота:

- `/catalog` — подбор обуви по каталогу с фильтрами  
- `/size_help` — помощь с выбором размера

---

## Команда `/catalog`

Диалог подбора обуви по каталогу. Реализован как конечный автомат с шагами:

1. **Выбор категории**

   Бот спрашивает:

   > Выберите категорию: мужская, женская, детская.

   Пользователь отвечает текстом:  
   `мужская` / `женская` / `детская`.

   Внутри это маппится на ключи `men`, `women`, `kids` (объект `CATEGORIES`).

2. **Фильтр по бренду (необязательный)**

   После успешного выбора категории бот спрашивает:

   > Категория: \<…\>.  
   > Хотите отфильтровать по бренду? Доступные бренды: Nike, Adidas, Puma.  
   > Если не важно — введите "нет".

   Варианты:

   - пользователь вводит один из брендов (`Nike`, `Adidas`, `Puma`) — в состоянии сохраняется выбранный бренд;
   - пишет `нет` — фильтр по бренду не применяется.

3. **Фильтр по цвету (необязательный)**

   Далее бот спрашивает:

   > Фильтровать по цвету? Доступные цвета: черный, белый, серый.  
   > Если не важно — введите "нет".

   Варианты:

   - пользователь вводит один из цветов — фильтр по цвету добавляется;
   - вводит `нет` — цвет игнорируется.

4. **Фильтр по размеру (необязательный)**

   Затем бот спрашивает:

   > Если хотите, укажите размер (например 40).  
   > Если не важно — введите "нет".

   Варианты:

   - пользователь вводит число (например `40`) — будет показана только обувь, где этот размер есть в списке размеров модели;
   - вводит `нет` — фильтр по размеру не используется;
   - если введено что-то нечисловое (`сорок` и т.п.) — бот просит ввести число или `"нет"`.

5. **Результат**

   По итогам всех шагов бот собирает фильтры:

   - категория (обязательная),
   - бренд (опционально),
   - цвет (опционально),
   - размер (опционально),

   и применяет их к каталогу моделей. Возможны два сценария:

   - Найдены подходящие модели:

     Бот отвечает:

     ```text
     Подходящие модели:

     Модель: Nike Air Max 270 (m1)
     Бренд: Nike
     Цвет: черный
     Размеры: 40, 41, 42, 43
     Наличие: ТЦ Европа, ТЦ Радуга

     ...
     ```

   - Ничего не найдено:

     ```text
     По выбранным фильтрам ничего не найдено.
     ```

Каждый пользователь хранится в отдельном состоянии по `chatId`, сценарии разных людей не пересекаются.

## Команда `/size_help`

Диалог помощи с размером обуви. Тоже реализован как конечный автомат.

Шаги:

1. **Пол / категория**

   Бот спрашивает:

   > Для кого обувь? Введите: мужская / женская / детская.

   Внутри выбирается нужная таблица размеров (`men`, `women`, `kids`).

2. **Длина стопы**

   Бот просит:

   > Введите длину стопы в сантиметрах (например 25.2).

   Введённое значение нормализуется (поддержка `25,2` → `25.2`), ищется интервал в таблице.  
   На выходе бот отдаёт рекомендуемый размер в разных системах (EU / RU / US).

3. **Код модели (опционально)**

   После подбора базового размера бот спрашивает:

   > Если хотите, введите код модели (например m1), чтобы учесть отзывы по посадке.  
   > Если не нужно — введите "нет".

   Варианты:

   - если пользователь вводит код существующей модели (`m1`, `w2`, `k3` и т.п.), бот дополняет ответ текстом отзыва (маломерит / большемерит / соответствует размеру);
   - если код не найден — бот сообщает, что отзывов нет;
   - если пользователь пишет `нет` — бот просто завершает диалог на расчёте размера.

В конце пользователь получает сводный ответ с рекомендованным размером и, при наличии, комментариями по конкретной модели.

## Технологии и структура проекта

- Node.js, модульный режим (`"type": "module"`)
- Телеграм-бот: `node-telegram-bot-api`
- Конфиги: `dotenv`
- Линтер: `eslint` (конфигурация на базе примера из задания)
- Тесты: `jest` с покрытием (`test:coverage`)

Структура:

```text
shoestore-bot/
  src/
    data/
      catalog.js      # данные по моделям, категориям, брендам, цветам, отзывам
      sizeTables.js   # таблицы размеров для men / women / kids
    utils/
      state.js        # in-memory состояние пользователей (FSM по chatId)
      validation.js   # валидации ввода
      catalog.js      # утилиты каталога: нормализация, фильтрация, форматирование
      catalogFlow.js  # конечный автомат для команды /catalog
      sizeHelp.js     # логика расчёта размера + отзывы по моделям
      sizeFlow.js     # конечный автомат для команды /size_help
    index.js          # точка входа
    server.js         # инициализация бота, регистрация обработчиков команд
  tests/
    catalog.test.js   # тесты утилит каталога
    flows.test.js     # тесты сценариев /catalog и /size_help
    sizeHelp.test.js  # тесты расчёта размеров
    state.test.js     # тесты хранилища состояний
  .env.example        # пример файла окружения
  .env                # ЛОКАЛЬНЫЙ токен бота (в репозиторий не коммитится)
  eslint.config.js
  jest.config.json
  package.json
  .gitignore
